# 실험노트 — 소개와 방법론

이 실험노트는 저장소에서 실제로 돌린 실험의 **방법·수치·재현 절차**를 기록합니다. 랩 노트의
목적은 마케팅 숫자가 아니라, 누구나 같은 커맨드로 **같은 결과에 도달**하게 만드는 것입니다.

    실험 01–07을 **하나의 이야기**로 읽고 싶다면 [스토리 아크 · 한눈에 보기](story-arc.md)를
    먼저 보세요 — 한 문장 논지, 여덟 실험 여정 표, 3막 내러티브, [실험 08 아레나](08-arena.md)
    에필로그, 정직함의 축을 담았습니다. 이 페이지(소개와 방법론)는 그 아래 **공통 방법론과
    지표 정의**를 다룹니다.

## 공통 방법론

- **오프라인·결정론적.** 네트워크·자격 증명·외부 호출이 없습니다. 합성 워크로드와
  결정론적 신호로 매번 동일하게 재현됩니다.
- **정직 라벨.** 모든 수치는 `labels.measured = false` — 합성 데이터에 대한 오프라인
  투영입니다. 측정된 절감이 아닙니다.
- **자리표시자 모델.** `mini-fast`, `swift-coder`, `balanced-pro`, `deep-reasoner`,
  `premium-max`는 모두 일반 자리표시자이며 특정 제품이 아닙니다.
- **재현성 계약.** 각 실험은 `expect` 하한을 두고, 오프라인 투영이 그 아래로 떨어지면
  실행이 실패합니다. 일부 실험은 `max_delta_pct` **상한**까지 두어(양방향 계약), 과장된
  **유령 절감**이 새어 나와도 실패합니다 — [실험 04](04-no-free-lunch.md) 참고. 나아가
  `max_tax_ratio`로 **팬아웃 세금 상한**까지 걸 수 있습니다 — [실험 06](06-fanout-dial.md) 참고.
  또한 `min_escalation_gain`으로 **에스컬레이션 이득 하한**을 두어, 관찰-후-에스컬레이션이
  단일 호출 라우팅보다 커버리지를 실제로 더 벌고 있는지 지킬 수 있습니다 —
  [실험 07](07-model-router.md) 참고.

## 비교 대상(arm) 정의

| arm | 선택 방식 | 성격 |
| --- | --- | --- |
| cost | 클래스별 가장 싼 후보 | 예시용 등가물 |
| balanced | 클래스별 중간 후보 | 예시용 등가물 |
| quality (naive) | 클래스별 가장 비싼 후보 | before(나이브) 기준선 |
| **비용 인지 라우팅** | 통과하는 가장 싼 모델 먼저, 실패 시 에스컬레이션 | 이 저장소의 방식 |

`cost`/`balanced`/`quality` arm은 투명한 **자리표시자 baseline**이며, 관리형 라우터의 내부
구현에 대한 주장이 아닙니다.

!!! tip "비용 × 커버리지 프런티어로 보기"
    [대시보드](../manual/dashboard.md)는 `all-mini`·`all-premium`·`cost-aware mix`·`all-ensemble`·
    `model_router` 다섯 전략을 **비용(가로) × 커버리지(세로) 산점도**로 그립니다. 오직 mix만
    좌상단 '둘 다 이기는' 코너(완전 커버리지 + 낮은 비용)에 위치하고, `all-ensemble`(전부 팬아웃)은
    커버리지 100%지만 **가장 비싼** 프런티어 밖 코너에, `model_router`(단일 호출 라우팅 레이어)는
    **낮은 커버리지**로 코너 밖 아래에 있는 것을 눈으로 확인할 수 있습니다 —
    [라이브 데모](https://hyeonsangjeon.github.io/foundry-cost-aware-model-routing/demo/?run=1).

## 지표

- **coverage** — 수용된(자체 신호가 깨끗한) 태스크 비율.
- **total_cost_usd** — 선택된 실행의 합산 비용(오프라인 투영).
- **delta_pct** — 나이브(quality arm) 대비 절감 비율.
- **spotlight ratio** — 대표 태스크에서 나이브/라우팅 비용 비율.

## 재현 방법

```bash
pip install -e .
cost-router experiment list
cost-router experiment run hero --json     # 기계가 읽는 전체 요약
```
