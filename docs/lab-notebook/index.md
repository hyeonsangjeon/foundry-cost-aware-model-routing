# 실험노트 — 소개와 방법론

이 실험노트는 저장소에서 실제로 돌린 실험의 **방법·수치·재현 절차**를 기록합니다. 랩 노트의
목적은 마케팅 숫자가 아니라, 누구나 같은 커맨드로 **같은 결과에 도달**하게 만드는 것입니다.

## 공통 방법론

- **오프라인·결정론적.** 네트워크·자격 증명·외부 호출이 없습니다. 합성 워크로드와
  결정론적 신호로 매번 동일하게 재현됩니다.
- **정직 라벨.** 모든 수치는 `labels.measured = false` — 합성 데이터에 대한 오프라인
  투영입니다. 측정된 절감이 아닙니다.
- **자리표시자 모델.** `mini-fast`, `swift-coder`, `balanced-pro`, `deep-reasoner`,
  `premium-max`는 모두 일반 자리표시자이며 특정 제품이 아닙니다.
- **재현성 계약.** 각 실험은 `expect` 하한을 두고, 오프라인 투영이 그 아래로 떨어지면
  실행이 실패합니다.

## 비교 대상(arm) 정의

| arm | 선택 방식 | 성격 |
| --- | --- | --- |
| cost | 클래스별 가장 싼 후보 | 예시용 등가물 |
| balanced | 클래스별 중간 후보 | 예시용 등가물 |
| quality (naive) | 클래스별 가장 비싼 후보 | before(나이브) 기준선 |
| **비용 인지 라우팅** | 통과하는 가장 싼 모델 먼저, 실패 시 에스컬레이션 | 이 저장소의 방식 |

`cost`/`balanced`/`quality` arm은 투명한 **자리표시자 baseline**이며, 관리형 라우터의 내부
구현에 대한 주장이 아닙니다.

## 지표

- **coverage** — 수용된(자체 신호가 깨끗한) 태스크 비율.
- **total_cost_usd** — 선택된 실행의 합산 비용(오프라인 투영).
- **delta_pct** — 나이브(quality arm) 대비 절감 비율.
- **spotlight ratio** — 대표 태스크에서 나이브/라우팅 비용 비율.

## 재현 방법

```bash
pip install -e .
cost-router experiment list
cost-router experiment run hero --json     # 기계가 읽는 전체 요약
```

각 실험 페이지 하단에 그 실험을 재현하는 정확한 커맨드가 있습니다.
