# 정직함 규약

이 프로젝트는 모든 수치·행위 주장에 **권한 라벨**을 유지합니다. 이 페이지는 무엇이 측정값이고
무엇이 투영인지, 무엇을 주장하고 무엇을 주장하지 않는지 명확히 합니다.

## 모델링 대 측정 경계

| 구분 | 이 저장소에서 | 라벨 |
| --- | --- | --- |
| 오프라인 before/after | 합성 데이터에 대한 투영 | `measured = false` |
| 실험 스포트라이트/절감률 | 합성 데이터에 대한 투영 | `measured = false` |
| 여러분 테넌트의 라이브 eval | 실제 측정 (범위: 측정한 워크로드) | `measured = true` |

**모든 번들 수치는 오프라인 투영입니다.** 측정된 절감은 여러분의 워크로드와 요율로 직접
돌려야만 얻어집니다. 그 라이브 실측 경로는 [라이브 실측 브릿지](manual/foundry-live.md)가
구현합니다 — 실제 Azure Model Router의 토큰 usage로 비용을 계산하고, 라이브 호출에만
`measured = true`를 부여합니다.

## 주장 권한 라벨

- **Tier 1 — 벤더 스펙.** 예: `retry-after-ms` 수용 신호, 문서화된 캐시 키 임계값, 공개 요율.
- **Tier 2 — 이 프로젝트의 추론/운영 정책.** 예: 시드 pass-rate/`$/resolved` 사전값,
  에스컬레이션 임계값, "값 X 이상에서만 앙상블" 규칙.

## 자리표시자 모델과 가격

- 모델 이름(`mini-fast`, `swift-coder`, `balanced-pro`, `deep-reasoner`, `premium-max`)은
  모두 일반 자리표시자이며 특정 제품이 아닙니다.
- `samples/pricing/illustrative.yaml`의 요율은 **더미 값**이며 누구의 공개 가격도 아닙니다.
  측정된 숫자를 원하면 이를 복사해 `your-tenant.yaml`(gitignored)에 실제 요율을 넣으세요.

## 청구 기준

현재 라우터는 미리 계산된 오프라인 신호에서 실행 하나를 선택하며, 그 신호 조회는 모델 호출이
아닙니다. 그래서 원장 비용은 `selected-execution-only` 기준입니다. 미래의 라이브 팬아웃 계층은
모든 패널/심판 호출을 별도로 회계해야 합니다. → [감사 원장](manual/ledger.md)

## 이것은 / 이것이 아닙니다

**이것은** 모델 선택을 감사 가능하고 비용 거버넌스가 붙은 결정으로 바꾸고, 그 결과를 라이브로
증명하는 오프라인 우선·배포 가능한 라우터입니다.

**이것이 아닌 것:**

- 공식 Foundry 가이드가 아닙니다.
- 측정된 SLA/처리량 모델이 아닙니다.
- 특정 절감 숫자에 대한 약속이 아닙니다.

절감은 전적으로 여러분의 워크로드 구성과 요율에 달려 있습니다 — 그래서 증명 단계를
**여러분의 테넌트에서** 돌리는 것입니다.

## 보안·재현성 규율

- 저장소에는 실제 키·토큰·연결 문자열·라이브 엔드포인트가 없습니다(검증 게이트가 스캔).
- `.env.sample`은 값 없는 자리표시자 이름만 담습니다.
- 모든 실험은 오프라인·결정론적으로 재현됩니다.
